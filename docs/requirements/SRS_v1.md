# 系统需求规格说明书 v1
## 第一章 项目概述

### 1.1 背景与意义
随着高等教育个性化痛点的加剧，学生在学习复杂概念时缺乏针对性的指导。
具体在计算机网络学习中，由于概念高度抽象，学生常产生多层次的误解，且传统工具缺乏协议仿真的直观呈现。
本系统提出一种基于大模型的多智能体解决方案，精准诊断误解并提供个性化矫正路径。

### 1.2 目标用户
主要：高校本科/高职计算机类专业学生
辅助：任课教师、助教

### 1.3 系统一句话定位
"能诊断学生为什么学不会TCP，并用协议仿真动画讲明白的智能学习教练"

### 1.4 与竞品的差异化
根据竞品分析报告：
1. 传统智学只停留在表面报错，我们能溯源至基础概念误解（三层诊断算法）。
2. 我们首创可视化参数化协议仿真，直观解释 TCP/UDP。
3. 学习路径可解释，告别黑盒盲目推荐。

## 第二章 功能性需求

### 2.1 核心功能清单
| 功能编号 | 功能名称 | 优先级 | 对应Agent |
|----------|---------|:------:|-----------|
| F01 | 对话式8维协议能力画像构建 | P0 | Profiler |
| F02 | 协议知识三库检索 | P0 | Retriever |
| F03 | 协议误解三层诊断 | P0 | Diagnosis |
| F04 | 协议仿真可视化 | P0 | Simulator |
| F05 | 5类多表征资源生成 | P0 | Generator |
| F06 | 可解释学习路径规划 | P0 | Planner |
| F07 | 误解对抗检验 | P1 | Challenger |
| F08 | 防幻觉与内容安全过滤 | P0 | 系统可信机制 |

### 2.2 8维画像详细需求
系统通过对话动态分析并记录学生的 8 个核心能力维度。通过 SQLite 和 JSON 进行持久化记录并在每次会话中更新。

### 2.3 三层诊断详细需求
第一层识别表面错误，第二层溯源基本概念盲点，第三层匹配常见经典错误模式。

## 第三章 非功能性需求

### 3.1 响应时间
- 普通对话：≤3秒首token到达
- 资源生成：≤15秒完整返回（附进度追踪）
- 协议仿真渲染：≤1秒首帧显示

### 3.2 内容安全
- 非计算机网络相关问题：拒绝回答并说明范围
- 超出知识库范围：强制输出"不确定声明"

## 第四章 系统架构概述

### 4.1 总体架构（C4 Model Level 1）
- 用户层：学生（主要）、教师（辅助）
- 应用层：Vue3前端
- 业务层：FastAPI后端 + 8 Agent协同
- 数据层：SQLite + Chroma + JSON知识图谱

### 4.2 8 Agent分工说明
Orchestrator 负责总控；Profiler 负责提取和更新画像；Retriever 负责多库知识检索；Diagnosis 负责误解推理；Simulator 负责协议渲染交互；Generator 生成表征资源；Planner 规划多步学习；Challenger 提供对抗测试。

### 4.3 三知识库结构
包含 course_docs (课程文档)、protocol_specs (协议标准)、misconceptions (误解特征库)。
