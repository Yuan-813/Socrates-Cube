# 核心创新点落地方案

## 一、三层认知诊断模型

### 设计理念
传统问答系统只判断对错，缺乏对"为什么答错"的分析。本系统参考教育心理学中的认知负荷理论与错误分析框架，将诊断分为三层递进：

| 层次 | 诊断目标 | 技术实现 |
|------|---------|---------|
| 表面错误层 | 识别当前答案的具体错误类型 | LLM 提示 + 关键词规则 |
| 根因分析层 | 定位缺失的前置知识节点 | 知识图谱反向遍历 + LLM 推理 |
| 误解模式层 | 归类学生错误的认知模式 | 误解库向量检索 + 模式匹配 |

### 落地代码位置
- `src/loopse/agents/diagnosis.py` — DiagnosisAgent，`diagnose()` 方法
- `config/prompts/diagnosis_*.txt` — 三层对应的提示词模板

## 二、知识图谱驱动的可解释路径规划

### 设计理念
区别于按章节顺序的线性推进，本系统基于知识图谱的有向依赖关系，动态生成"最短补偿路径"：先补弱项前置知识，再推进目标知识点，每步给出可追溯的推荐理由。

### 知识图谱结构
- 20 个知识节点（kn_001 ~ kn_020），覆盖计算机网络五大章节
- 21 条有向依赖边，反映"学 A 需先掌握 B"的前提关系
- 算法：拓扑排序（Kahn算法）+ DFS 前置依赖搜索

### 落地代码位置
- `data/knowledge_graph.json` — 图结构数据
- `src/loopse/kb/knowledge_graph.py` — KnowledgeGraph 类，`get_all_prerequisites()`, `topological_sort()`, `find_weak_prerequisites()`
- `src/loopse/agents/path_planner.py` — PathPlannerAgent，`plan()` 方法

## 三、八维学习画像动态追踪

### 八个维度
1. 网络分层认知（layered_model_cognition）
2. 协议流程记忆（protocol_flow_memory）
3. 报文格式理解（packet_format_understanding）
4. 协议间关系（inter_protocol_relation）
5. 故障排查逻辑（fault_diagnosis_logic）
6. 实践操作能力（practical_operation）
7. 认知风格偏好（cognitive_style）
8. 常见误解模式（misconception_pattern）

### 追踪机制
每轮对话后 ProfilerAgent 调用 LLM 提取增量 delta，将变化量叠加到画像数据库，确保画像随学习过程持续精化而非覆盖。

### 落地代码位置
- `src/loopse/agents/profiler.py` — ProfilerAgent，`update_from_dialogue()` 方法
- `frontend/src/stores/userStore.ts` — radarScores 计算属性

## 四、实时 SSE 多事件流架构

### 设计理念
多 Agent 协作过程对用户不透明是传统系统的痛点。本系统将每个 Agent 的启动、结果、工具调用全部封装为 SSE 事件实时推送，前端逐事件响应，实现"思考过程可视化"。

### SSE 事件类型
| 事件 | 含义 |
|------|------|
| agent_start / agent_end | Agent 开始/完成 |
| tool_call | 调用外部工具（检索、数据库） |
| token | LLM 生成字符流 |
| diagnosis | 诊断结果对象 |
| resource | 生成的学习资源 |
| path_update | 更新学习路径 |
| done / error | 流结束/异常 |

### 落地代码位置
- `src/loopse/agents/orchestrator.py` — `async_stream_reply()` 异步生成器
- `src/loopse/api/chat.py` — EventSourceResponse 封装
- `frontend/src/composables/useSSE.ts` — `useChatSSE()` 前端消费
