# 测试说明书 v1.0

## 一、概述

本文档说明 Socrates-Cube（苏格拉底问答学习系统）的测试策略、测试环境要求及各模块验收标准，适用于软件杯 A3 赛项评审。

---

## 二、测试环境

| 项目 | 要求 |
|------|------|
| Python | 3.10+ |
| 虚拟环境 | d:\software_APP\.venv\ |
| 测试框架 | pytest 8.2.0 + pytest-asyncio 0.23.7 |
| 前端 | Node.js 18+，Vite 5.x |
| LLM | 讯飞星火（无密钥时自动 mock 模式） |
| 数据库 | SQLite（edu_agent.db，测试时自动初始化） |

---

## 三、测试分层

### 3.1 单元测试（tests/unit/）

| 测试文件 | 覆盖目标 | 关键用例 |
|---------|---------|---------|
| test_diagnosis_agent.py | DiagnosisAgent 三层诊断 | 表面错误检测、根因分析、模式匹配、完整流水线 |
| test_resource_generator.py | ResourceGeneratorAgent | doc/exercise/code 三类生成、返回结构完整性 |
| test_student_profile_schema.py | StudentProfile 数据结构 | 字段类型校验、默认值 |

运行方式：
```bash
cd d:\software_APP\Socrates-Cube
.\.venv\Scripts\activate
pytest tests/unit/ -v
```

### 3.2 集成测试（tests/）

| 测试文件 | 覆盖目标 |
|---------|---------|
| test_spark_sse.py（outputs/） | SSE 流式接口端到端 |

### 3.3 接口测试

启动后端服务后访问 `http://localhost:8000/docs` 使用 Swagger UI 手动验收全部 REST 接口。

---

## 四、LLM Mock 模式说明

当环境变量 `SPARK_APP_ID` 未设置时，系统自动切换到 Mock 模式：
- `chat()` 返回固定模板回复
- `async_stream_chat()` 按字符流逐字 yield
- 不依赖外部网络，适合 CI 环境和离线演示
