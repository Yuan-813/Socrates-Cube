# 项目定位说明

> 版本：v0.1.0 | 最后更新：2026-05-08 | 知识库版本：谢希仁《计算机网络》第 8 版

---

## 1. 系统一句话定位

**Socrates-Cube** 是面向高校《计算机网络》课程的多智能体自适应学习系统，通过构建学生六维画像动态驱动 6 个专业 Agent，实现"诊断→检索→生成→规划"全链路个性化辅学闭环。

---

## 2. 课程范围

知识库版本锁定为**谢希仁《计算机网络》第 8 版**，系统聚焦以下 5 个核心章节：

| 章节 | 主题 | 重点知识点 |
|------|------|-----------|
| 第 1 章 | 概述 | 分层体系结构、TCP/IP 模型、协议与服务 |
| 第 3 章 | 传输层 | TCP/UDP、三次握手四次挥手、拥塞控制、可靠传输 |
| 第 4 章 | 网络层 | IP 地址、子网划分、路由协议（RIP/OSPF/BGP）、NAT |
| 第 5 章 | 数据链路层 | MAC 地址、以太网、滑动窗口、差错控制 |
| 第 6 章 | 应用层 | HTTP/HTTPS、DNS、FTP、SMTP |

---

## 3. 六个 Agent 清单

| Agent 名称 | 一句话职责 |
|-----------|-----------|
| **Orchestrator**（调度器） | 接收用户输入，分析意图，协调其余 5 个 Agent 的执行顺序与数据流转 |
| **Profiler**（画像器） | 根据对话历史和答题记录，动态更新并维护学生六维画像 JSON |
| **Retriever**（检索器） | 对接 ChromaDB 向量库，从教材和讲义中检索与当前问题最相关的知识片段 |
| **Diagnosis**（诊断器） | 分析学生错误模式，定位薄弱知识点，输出结构化的学习障碍报告 |
| **Generator**（生成器） | 根据画像和诊断结果，生成个性化讲解、习题、记忆助手等学习材料 |
| **Planner**（规划器） | 基于学习进度和薄弱点，生成个性化学习路径规划与阶段目标 |

---

## 4. 三个核心创新点

1. **六维动态画像**：知识储备、认知风格、学习进度、协议理解、动手能力、常见错误六个维度协同驱动内容生成，实现真正意义上的个性化自适应。

2. **多 Agent 流水线架构**：6 个 Agent 通过结构化状态（GraphState）串联，支持 SSE 流式输出，用户可实时看到每个 Agent 的推理过程，增强可解释性。

3. **领域知识增强生成（RAG）**：以谢希仁第 8 版教材为权威知识源，结合向量检索防止模型幻觉，所有生成内容均可追溯知识库原文。
